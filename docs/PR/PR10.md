# PR10: 長時間使用時のGPUメモリ枯渇によるネイティブクラッシュを修正

## 概要

常駐モードで多数回連続して文字起こしを行うと、CTranslate2/CUDAのGPUメモリが
累積的に枯渇し、Pythonの例外ハンドラをすり抜けるネイティブクラッシュが発生する。

再現条件: large-v3 + cuda + float16 で約25回以上の連続文字起こし後にクラッシュ。

## 対応方針

1. `gc.collect()` を各 transcribe 後に呼び出してPython GCを促進
2. N回ごとに WhisperModel を再初期化してGPUメモリを解放（デフォルト: 30回）
3. 設定 `WHISPER_REINIT_INTERVAL` で再初期化間隔を制御可能にする

## 作業内容

- [ ] `voice_paste/transcription/whisper_transcriber.py` — セッションカウンター追加、N回ごとにモデル再初期化、各transcribe後にgc.collect()
- [ ] `voice_paste/config.py` — `WHISPER_REINIT_INTERVAL` 設定追加
- [ ] `.env.sample` — `WHISPER_REINIT_INTERVAL` のデフォルト記載追加

## テスト

- [ ] 30回以上連続文字起こしして落ちないこと
- [ ] 再初期化後も正常に文字起こしできること
- [ ] `WHISPER_REINIT_INTERVAL=0` で再初期化無効にできること
